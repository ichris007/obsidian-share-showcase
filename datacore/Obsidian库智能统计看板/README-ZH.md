## VaultInsight - Obsidian 知识库智能统计看板

[English](./README.md) | [中文](./README-ZH.md)

### 1. 📖 脚本概述

这是一个基于 Obsidian Datacore插件的 Vault 统计看板脚本，为您提供了一套完整的笔记库数据监控与分析工具。通过可视化面板，您可以实时掌握知识库的运行状态，包括文件数量、标签统计、项目分布、任务进度等关键指标。

### 2. ✨ 核心功能

#### 2.1 📊 核心指标总览
- **使用天数**：基于笔记的最早创建日期自动计算
- **文件总数**：支持 MD、Canvas、Excalidraw、图片及杂项文件统计
- **标签数量**：自动扫描并统计所有标签（按一级分类去重）
- **项目总数**：智能识别 Frontmatter 中的项目字段
- **任务总数**：统计所有待办、进行中、已完成和已取消任务

#### 2.2 🎯 项目分布看板
- 自动识别包含 `project` / `projects` / `parent_project` 等属性字段的笔记
- 根据属性 `status` 字段智能归类项目状态
- 为不同状态自动匹配色彩方案（进行中、已完成、延期、暂停等）
- 支持项目专项路径隔离（包含/排除指定目录）

#### 2.3 📋 任务看板
- 实时统计任务状态分布：待办、进行中、已完成、已取消等
- 基于 Dataview 任务语法解析（Datacore解析任务不成功后采用的方案）
- 自动排除指定目录中的任务

#### 2.4 🛠️ 交互式配置中心
- **可视化配置面板**：无需修改代码即可调整所有统计参数
- **动态路径管理**：
  - 全局排除目录（如 Templates、.trash 等）
  - 项目统计专属包含/排除目录（支持父子目录精确隔离）
- **文件类型筛选**：自定义哪些类型的文件计入总数
- **面板显示控制**：独立控制每个子模块的显隐
- **配置快照导出**：一键复制当前配置的完整代码，方便固化到脚本中

### 3. 🎨 特色亮点

#### 3.1 智能色彩映射
- 项目状态自动匹配语义化颜色方案
- 任务状态使用差异化视觉标识
- 未识别状态自动生成和谐色值

#### 3.2 响应式设计
- 完美适配 Obsidian 深色/浅色模式

- 自适应布局，移动端友好

#### 3.3 精准路径隔离
- 支持多层级目录的精确包含与排除
- 项目统计可独立于全局配置
- 父子目录关系的智能阻断机制

### 4. 🚀 使用方法

#### 4.1 前置条件
1. 确保已安装并启用 [Datacore](https://github.com/blacksmithgu/datacore) 插件
2. 安装 [Dataview](https://github.com/blacksmithgu/obsidian-dataview) 插件以支持完整任务统计功能

#### 4.2 安装步骤

1. 在 Obsidian 中创建一个新的笔记文件
2. 将本脚本的完整代码复制到该笔记中（[脚本]()）
3. 确保笔记语言设置为 `datacore`（确保代码开头添加 ````datacorejsx` 代码块标记）
4. 切换到阅读模式或退出代码编辑区即可看到统计面板

#### 4.3 快速配置指南

##### 4.3.1 基础配置（通过界面操作）
1. 点击统计面板右上角的 **⚙️ 统计设置** 按钮
2. 在展开的配置面板中调整各项参数：
   - **主卡片标题**：自定义看板名称
   - **全局排除目录**：添加/删除需要忽略的文件夹
   - **项目统计路径**：设置项目专项包含/排除目录
   - **文件类型筛选**：勾选需要计入总数的文件类型
   - **子面板控制**：显示/隐藏各个功能模块
3. 配置即时生效，无需刷新页面

##### 4.3.2 固化配置（将配置写入代码）
1. 在配置面板底部点击 **📋 复制固化配置代码**
2. 打开脚本源文件
3. 将复制的内容替换顶部的 `CONFIG` 对象
4. 保存文件即可永久生效

#### 4.4 配置参数详解

```javascript
const CONFIG = {
  // 显示控制
  showTitle: true,              // 是否显示标题
  titleText: "Vault 运行数据概览", // 自定义标题
  
  // 全局排除目录
  excludedFolders: ["Templates", ".trash", ".obsidian", ".datacore"],
  
  // 文件类型包含控制
  includeFilesForTotal: {
    canvas: false,       // Canvas 白板文件
    excalidraw: false,   // Excalidraw 绘图
    images: false,       // 图片文件
    other: false         // 其他类型文件
  },
  
  // 项目统计路径隔离
  projectIncludedFolders: [],   // 项目包含目录（空=全库）
  projectExcludedFolders: [],   // 项目排除目录
  
  // 子面板显隐控制
  showDetailsPanel: true,       // 文件明细面板
  showProjectPanel: true,       // 项目分布面板
  showTaskPanel: true,          // 任务分布面板
  showFooterPanel: true         // 底部历史时间
};
```

### 5. 📝 使用示例场景

#### 5.1 场景一：排除所有临时文件
将 `.trash`、`Templates`、`Daily Notes` 添加到全局排除目录，统计结果将自动过滤这些文件夹中的内容。

#### 5.2 场景二：专注项目文件夹统计
在项目专项配置中：
- **包含目录**：`["Projects/Active", "Projects/Archived"]`
- **排除目录**：`["Projects/Archived/2023"]`

这样只会统计 Active 和 Archived 文件夹中的项目，但排除 2023 年的归档项目。

#### 5.3 场景三：自定义看板展示
- 关闭文件明细面板（`showDetailsPanel: false`）
- 只保留核心指标、项目分布和任务看板
- 打造专属于项目管理的数据仪表盘

### 6. 🔧 兼容性与依赖

- **核心依赖**：Datacore 插件（必需）
- **可选依赖**：Dataview 插件（必需，用于增强任务统计）
- **兼容版本**：Obsidian v1.12.7（其它版本未测试）
- **主题适配**：自动适配所有 Obsidian 主题

### 7. 📌 注意事项

1. **性能优化**：对于大型 Vault（5000+ 文件），建议合理设置排除目录以减少计算量
2. **项目识别**：确保笔记 Frontmatter 中包含 `project` 或 `status` 字段
3. **任务统计**：仅当启用 Dataview 插件时，任务统计功能才完整可用
4. **路径格式**：文件夹路径使用相对路径，如 `"Projects/Active"`，不要包含前导斜杠

### 8. 🐛 问题排查

| 问题 | 解决方案 |
|------|----------|
| 统计面板不显示 | 确认文件语言设置为 `datacore`，且已启用 Datacore 插件 |
| 项目数为 0 | 检查笔记 Frontmatter 是否包含 `project` 字段 |
| 任务统计不准确 | 安装并启用 Dataview 插件，刷新面板 |
| 某个文件夹无法排除 | 检查路径名称是否完全匹配（区分大小写） |
| 配置保存后不生效 | 复制配置代码并替换脚本顶部的 CONFIG 对象 |
